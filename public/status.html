OpenClaw — Autonomous Lead Agent
Background agent running 24/7. No humans involved.
OpenClaw continuously scrapes Google Maps and local business directories, visits each website, and scores it 0-100 on quality. Businesses with broken sites, missing sites, or score ≤40 get flagged as hot leads and automatically fired into the FreshFacing pipeline. The pipeline builds a custom site, takes a screenshot, adds the owner to Instantly, and sends a personalized email with their redesign inline. The business owner gets an email that looks hand-crafted. We never touched anything.

How the autonomous loop works
1
OpenClaw discovers businesses
Scrapes Google Maps, Yelp, and directories for local businesses in target categories. Dental, legal, medical, restaurants, real estate. Stores URL + contact data.
2
Agent scores each website 0-100
Visits every URL. Checks for broken DNS, missing site, no HTTPS, poor mobile, outdated design. Outputs a quality score. 83% of scanned businesses score 0 (no site or error).
3
Hot leads fired into pipeline
Score ≤40 triggers automatic webhook to Railway. Pipeline queues the job, scrapes their existing site (if any), runs Gemini 10-step analysis, and generates a full replacement in Claude Opus 4.6.
4
Site built and deployed
Custom HTML site ships to Cloudflare Pages at freshfacing.com/{slug}. Watermarked. Apify takes a 1440px screenshot, PIL compresses it to ~200KB, uploaded to Supabase Storage.
5
Owner added to Instantly campaign
Lead added with companyName, category, area, and screenshot_url pre-filled. Instantly sends the 1-step email sequence: personalized copy + inline screenshot of their new site.
6
Owner pays, watermark removed
Stripe Checkout → webhook → Railway /fulfill-payment. HTML pulled from Supabase Storage, watermark stripped, site redeployed clean. Confirmation email sent. $20/month recurring. We never log in.

06
Open Items
01
Zero auth on /fulfill-payment CRITICAL
Anyone with the URL can trigger watermark removal, spam confirmation emails, and exfiltrate HTML. Easy to find via Stripe webhook logs.
→ FULFILL_SECRET env var on Railway + Vercel · validate X-Fulfill-Secret header via hmac.compare_digest()
02
Zero auth on /api/dashboard CRITICAL
Returns all customer emails, site URLs, Stripe revenue, every active subscription — completely public with no token required.
→ DASHBOARD_SECRET server-side · x-dashboard-key header check · NEXT_PUBLIC_DASHBOARD_KEY for client fetch
03
No idempotency on fulfillment
Stripe retries webhooks. session_id is received but never stored. Same payment can trigger double watermark removal + double confirmation emails to a real customer.
→ store fulfilled_session in Supabase notes · check before processing · skip if already done
04
Clay webhook fires before email verification
31/35 webhooks arrived with verified=false and no email. Clay fires on row creation, not after enrichment. Massively limiting throughput.
→ add Clay filter: only fire when "Verified - Email - People" = true AND email is not empty
05
_normalize_url() lstrip() bug
lstrip("www.") strips any leading chars in {w,o,m,.} — not the prefix "www.". westoverdentistry.com → estoverdentistry.com. Wrong cache hits, duplicate Supabase rows.
→ replace lstrip("www.") with removeprefix("www.") in pipeline.py
